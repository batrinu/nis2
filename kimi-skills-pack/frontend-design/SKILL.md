# Frontend Design: Distinctive & Production-Grade

This skill creates frontend interfaces that look like they were designed by a talented human designer, not generated by an AI. Every output should be visually striking and memorable.

## Design Thinking Process

Before writing any CSS or markup, commit to a BOLD aesthetic direction:

1. **Purpose** — What problem does this interface solve? Who uses it?
2. **Tone** — Pick a strong direction. Examples:
   - Brutally minimal (stark, lots of whitespace, monospace type)
   - Maximalist chaos (dense, colorful, layered, energetic)
   - Retro-futuristic (CRT glow, scanlines, neon accents)
   - Organic/natural (earthy tones, rounded shapes, flowing layouts)
   - Luxury/refined (thin serifs, gold accents, dark backgrounds)
   - Playful/toy-like (bright primaries, chunky shapes, bouncy animations)
   - Editorial/magazine (strong grid, dramatic type hierarchy, hero images)
   - Brutalist/raw (system fonts, harsh borders, no decoration)
   - Art deco/geometric (symmetry, gold lines, rich patterns)
   - Industrial/utilitarian (monospace, status indicators, dense data)
   - Glassmorphism (frosted glass, blur, layered transparency)
   - Neo-Memphis (geometric shapes, bold colors, playful patterns)
3. **Signature element** — What ONE thing will make this unforgettable?
4. **Constraints** — Framework, performance, accessibility requirements

**CRITICAL**: Choose a clear direction and execute it with conviction. Bold maximalism and refined minimalism both work — the key is intentionality, not timidity.

## Typography Rules

Typography is the #1 differentiator between generic and premium interfaces.

**DO:**
- Use distinctive, characterful fonts from Google Fonts or similar CDNs
- Pair a bold display font with a refined body font
- Create clear hierarchy: display → heading → subheading → body → caption
- Use dramatic size contrasts (e.g., 72px heading + 16px body)
- Letter-spacing adjustments for headings (slightly tighter) and all-caps text (slightly wider)

**NEVER use these generic fonts:**
- Inter, Roboto, Arial, Helvetica, system-ui, sans-serif (as primary)
- These are the hallmark of "AI slop" design

**Good font pairing examples:**
- Display: Playfair Display / Body: Source Sans 3
- Display: Space Mono / Body: DM Sans
- Display: Fraunces / Body: Commissioner
- Display: Syne / Body: Outfit
- Display: Clash Display / Body: Satoshi
- Display: Cabinet Grotesk / Body: General Sans

## Color & Theming

**DO:**
- Define a cohesive palette using CSS custom properties
- Use a dominant color with 1-2 sharp accent colors
- Create depth with subtle shade variations (not just opacity)
- Consider the emotional weight of your palette
- Support dark mode with `prefers-color-scheme` or a manual toggle

**DON'T:**
- Use purple gradients on white backgrounds (the #1 AI cliché)
- Distribute colors evenly — dominant + accent beats equal distribution
- Use pure black (#000) for text — use dark gray (#1a1a1a or similar)
- Rely solely on color for meaning (accessibility)

**CSS variable pattern:**
```css
:root {
  --color-bg: #faf9f6;
  --color-surface: #ffffff;
  --color-text: #1a1a2e;
  --color-text-muted: #6b7280;
  --color-accent: #e63946;
  --color-accent-hover: #c1121f;
  --radius-sm: 4px;
  --radius-md: 8px;
  --radius-lg: 16px;
  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 32px;
  --space-xl: 64px;
}
```

## Layout & Composition

- **Break the grid** — Asymmetric layouts, overlapping elements, diagonal flow
- **Generous whitespace** — Let elements breathe. Cramped = amateur
- **Unexpected sizing** — Hero sections that take 80vh, tiny elegant footers
- **Visual rhythm** — Alternate between dense and sparse sections
- **Z-depth** — Use shadows, layering, and backdrop-blur to create depth

## Motion & Animation

**High-impact animations (do these first):**
- Page load: staggered reveal of elements using `animation-delay`
- Scroll-triggered entrances (use Intersection Observer)
- Hover states that transform meaningfully (not just color change)
- Page transitions between routes

**Micro-interactions (add if time allows):**
- Button press feedback (scale + shadow change)
- Input focus glow/border animation
- Toggle switches with spring physics
- Loading skeleton shimmer effects
- Success/error state transitions

**Implementation:**
```css
/* Staggered entrance */
.reveal { opacity: 0; transform: translateY(20px); animation: fadeUp 0.6s ease forwards; }
.reveal:nth-child(1) { animation-delay: 0.1s; }
.reveal:nth-child(2) { animation-delay: 0.2s; }
.reveal:nth-child(3) { animation-delay: 0.3s; }

@keyframes fadeUp {
  to { opacity: 1; transform: translateY(0); }
}
```

## Backgrounds & Atmosphere

Don't default to flat solid colors. Create atmosphere:
- Gradient meshes (multi-point radial gradients)
- Subtle noise/grain textures (CSS or SVG filter)
- Geometric patterns (repeating SVG backgrounds)
- Layered transparencies
- Dramatic shadows and light sources
- Decorative SVG shapes as accents

## Anti-Patterns: The "AI Slop" Checklist

If your output has 3+ of these, redesign:
- [ ] Inter or Roboto font
- [ ] Purple-to-blue gradient
- [ ] Perfectly centered everything
- [ ] Uniform 12px border-radius on all elements
- [ ] Generic stock-photo placeholder circles
- [ ] "Lorem ipsum" text
- [ ] Identical card layouts in a 3-column grid
- [ ] No hover/focus states
- [ ] No animations or transitions
- [ ] White background with gray text only

## Final Quality Check

Before delivering, verify:
1. Does this look like it was designed by a talented human? (not a template)
2. Is there a clear visual hierarchy? (you know where to look first)
3. Is there a signature element? (something memorable)
4. Are interactions smooth? (hover, focus, transitions)
5. Does it work on mobile? (375px viewport)
6. Is the typography distinctive? (not Inter/Roboto/Arial)
7. Are colors cohesive? (palette, not random)

Remember: You are capable of extraordinary creative work. Don't settle for generic. Every project deserves a unique visual identity.
